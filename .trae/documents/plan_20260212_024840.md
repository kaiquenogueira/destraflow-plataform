Vou remover a flag `--skip-generate` que está causando erro no seu terminal. Embora a documentação antiga mencionasse essa flag, nas versões mais recentes do Prisma CLI ela foi removida ou está causando conflito em alguns ambientes, e o comportamento padrão do `db push` já não gera o cliente automaticamente a menos que configurado especificamente.

### Plano de Correção

1.  **Corrigir Scripts de Sincronização (`src/actions/tenant-sync.ts` e `scripts/sync-tenants.ts`)**:
    *   Remover a flag `--skip-generate` dos comandos `prisma db push`.
    *   Isso resolverá o erro `unknown or unexpected option: --skip-generate` que você viu.

2.  **Garantir a Integridade do Cliente (`package.json`)**:
    *   Como vamos remover o `skip-generate`, precisamos garantir que o cliente principal (gerado a partir de `schema.prisma` completo) seja restaurado após as operações de manutenção.
    *   Atualizarei os comandos `db:push:crm` e `db:push:tenant` para rodarem `prisma generate` (usando o schema principal) logo após o push.
    *   Isso garante que, mesmo que o push gere um cliente parcial temporário (o que não deve acontecer, mas é bom prevenir), o comando final sempre restaura o cliente completo da aplicação.

### Resumo da Fronteira Definida

*   **Aplicação (CRM/Admin)**: Usa `prisma migrate` no banco principal.
*   **Tenants (Clientes)**: Usam `prisma db push` (sem migrações SQL) disparado pela aplicação ou scripts de manutenção.
